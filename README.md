# Medical\_database



\# Вашему вниманию представлено приложение "Регистратура", которое реализует взаимодействие пользователя с базой данных SQLite, через удобный и понятный интерфейс.

\## Принцип работы приложения

При запуске приложения пользователь выбирает в меню раздел "Файл" -> "Подключиться к БД". Далее идет подключение к базе и индикация основных виджетов приложения. Кнопки становятся доступны пользователю, и на экране выводятся данные из файла БД в табличном виде, в statusbox выводится дополнительная информация. 

\## Основные вкладки

\-  "Исследования" - представлена реляционная таблица из БД, поля которой NAME\_DOCTOR и NAME\_PATIENT  связаны с таблицами "Пациенты" и "Доктора".

\-  "Пациенты" - представлена таблица "Пациенты"

\-  "Доктора" - представлена таблица "Доктора"



* При работе с вкладкой "Исследования", при добавлении записи пользователь должен обязательно заполнить поля NAME\_DOCTOR и NAME\_PATIENT (при нажатии на ячейку выводится список докторов и пациентов из которых нужно выбрать требуемого). Остальные поля заполнить пользователь может в порядке необходимости. Для изменения данных нужно выбрать ячейку и изменить её.
* При работе с вкладкой "Пациенты", при добавлении записи пользователь должен заполнить ячейку NAME\_PATIENT для корректного отображения данных в реляционной базе "Исследования".Поле "CARDIO" отображает кардиограмму пациента. При нажании на ячейку из поля "СRDIO"  открывается дополнительное окно больших размеров для детального рассмотрения кардиограммы. Для ввода кардиограммы или замены текущей кардиограммы, нужно свернуть текущее окно с открытой кардиограммой, щелкнуть дважды на ячейке и ввести цифру (от 1 до 3, тк пока только 3 рисунка есть). А если поле ячейки итак было пустое, то ничего выводится не будет на экран, пока пользователь в не укажет цифру в ячейке.

   Для изменения других данных нужно выбрать ячейку и изменить её.

* При работе с вкладкой "Доктора", при добавлении записи пользователь должен заполнить ячейку NAME\_DOCTOR для корректного отображения данных в реляционной базе "Исследования". Остальные поля заполнять можно по мере необходимости.
* Для удаления любых записей, нужно выбрать ячейку и нажать на кнопку "Удалить", будет удалена строка, на которой находится выбранная ячейка.



\### Взаимодействие с пользователем через кнопки

\-  "Добавить" создание записи в БД

\-  "Удалить" удаление записи из БД

\-  "Сортировать" сортировка данных представленных в таблице ( по возрастанию, по убыванию) в соответствии с тем полем сортировки, которое пользователь выбрал в combobox

\-  "Отмена сортировки" отмена сортировки



\## Взаимодействие с пользователем через поле LineEdit

\- Поиск данных в таблицах осуществляется путём ввода данных в лайнедит и выбора соответствующей категории поиска (combobox - в нем представлены имена столбцов из БД).

Для удобства поиска выбрана стратегия частичного совпадения данных. Как только введенный символ будет совпадат с табличным значением, то ячейка таблицы выбед автоматически выделена цветом, а в нижней части окна приложения (Statusbox) будет выведено сообщение с количеством совпадающих записей.



\## Взаимодействие с пользователем через всплывающие информационные окна.

\- Подсказка при удалении данных

\- Предупреждение при удалении данных

\- Предупреждение при добавлении данных
